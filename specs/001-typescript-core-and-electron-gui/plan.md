# Implementation Plan: TypeScript Core + Electron GUI

**Branch**: `001-typescript-core-and-electron-gui` | **Date**: 2026-03-28 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/001-typescript-core-and-electron-gui/spec.md`

## Summary

Port csvons Go core validation library to TypeScript and build an Electron desktop application, all inside `gui/electron/` as a standalone folder. The TS core lives in `src/core/` within the Electron project. Uses **bun** as package manager. Generators replace Go channels for lazy field value extraction. Validation errors are collected (not panic-on-first). papaparse provides RFC 4180-compliant CSV parsing. React 19 + Vite powers the renderer.

## Technical Context

**Language/Version**: TypeScript 5, Node.js (Electron 41 runtime)
**Package Manager**: bun
**Primary Dependencies**: papaparse (CSV), React 19 (renderer), electron 41, vitest (testing)
**Storage**: JSON file for local state (recent paths)
**Testing**: vitest with istanbul coverage (>=80% threshold)
**Target Platform**: macOS/Windows/Linux desktop (Electron)
**Project Type**: standalone desktop-app with embedded validation library
**Performance Goals**: 100k row CSV validation < 2s
**Constraints**: Linear memory scaling, no unbounded row accumulation
**Scale/Scope**: Single standalone package, ~15 source files, ~15 test files
